# Strivory 产品需求文档（PRD）

| 项目 | 内容 |
| --- | --- |
| 产品名称 | Strivory |
| 文档版本 | v1.0 |
| 状态 | 已确认，作为第一版开发基线 |
| 平台 | iPhone / iPad（以 iOS 为主） |

## 1. 产品定义

Strivory 是一款读取 Apple 健康运动记录，并将其沉淀为可回望、可保存、可分享的「运动年历」的本地优先 App。

产品不以训练计划、社交或实时数据分析为核心；第一版的核心价值是让用户以年度日历的形式看见一生的运动轨迹，并导出为精美图片。

## 2. 产品目标与范围

### 2.1 第一版目标

- 从 HealthKit 读取 Workout 记录。
- 支持 CSV 导入，用于补全或修正历史运动日期。
- 按自然年生成「一日一格」的运动年历。
- 每年只显示五种颜色：四个该年度最主要的运动大类，以及“其他”。
- 支持单年图片和多年纵向长图导出。
- 全部原始运动数据与统计结果默认只在用户设备本地处理和保存。

### 2.2 第一版不包含

- 训练计划、课程、饮食、排行榜、社交关系或 AI 教练。
- 创建、修改或回写 HealthKit Workout。
- 账号体系、云端同步、服务端数据分析。
- Apple Watch 独立应用。
- 心率、配速、分段等专业训练分析。

## 3. 关键定义

| 名称 | 定义 |
| --- | --- |
| 有效 Workout | 单条 HealthKit Workout 时长大于或等于 10 分钟。 |
| 活动日 | 一个自然日内存在至少一条有效 Workout，或存在一条有效 CSV 导入记录。一天最多计为一个活动日。 |
| 每日主运动 | 当天各运动大类累计时长最高的大类。 |
| 运动大类 | 由细粒度 Workout 类型映射而来的稳定类别，例如跑步、游泳、力量。 |
| 年度前四 | 该年度中作为“每日主运动”出现天数最多的四个运动大类。 |
| 其他 | 当年未进入年度前四的所有运动大类的合计；不是仅指未知类型。 |

## 4. 数据来源

### 4.1 HealthKit Workout

用户授权后，App 读取 Apple 健康中的 Workout 数据。数据可能来自 Apple Watch、iPhone 或已向 Apple 健康写入记录的第三方运动 App。

每条记录至少使用以下字段：

| 字段 | 用途 |
| --- | --- |
| 开始时间、结束时间 | 确定归属日期与年份。 |
| Workout 类型 | 映射为运动大类。 |
| 时长 | 筛选小于 10 分钟的记录，并决定每日主运动。 |
| 活动能量 | 同日时长相同的最终比较依据。 |
| 距离 | 预留给日详情与后续统计，不影响第一版年历颜色。 |
| 来源 App / 设备 | 用于识别潜在重复数据。 |

处理规则：

1. 单条时长少于 10 分钟的 Workout 忽略。
2. 跨越午夜的 Workout 归属训练开始日期。
3. 日期按用户当前选择的本地时区计算。
4. 对明显重复的 Workout（时间高度重叠、类型与时长相近、来源不同）进行去重；保留可信的单条记录。
5. HealthKit 数据变化后，年历和统计应可重新计算。

### 4.2 CSV 导入

CSV 是 HealthKit 的补充数据源，主要用于补录历史记录。导入不会将记录写回 Apple 健康。

#### 4.2.1 文件格式

首行必须包含表头；支持英文或中文表头。

```csv
date,workout_type
2025-01-01,户外跑步
2025-01-02,力量训练
2025-01-04,泳池游泳
```

```csv
日期,运动类型
2025-01-01,跑步
2025-01-02,HIIT
```

| 字段 | 要求 |
| --- | --- |
| `date` / `日期` | 必填；格式严格为 `YYYY-MM-DD`；不接受未来日期。 |
| `workout_type` / `运动类型` | 必填；支持中文、英文及已配置别名。 |
| 编码 | UTF-8，兼容 UTF-8 BOM。 |

CSV 不含时长；每条合法记录被视作一个已确认的活动日记录，不应用“少于 10 分钟”筛选。

#### 4.2.2 导入与冲突规则

- 默认模式为“补全缺失日”：若同一天已有有效 HealthKit Workout，则保留 HealthKit 的每日主运动，CSV 行标为冲突但不写入统计。
- 可选模式为“CSV 覆盖同日记录”：该日期以 CSV 指定类型作为每日主运动。
- 一个 CSV 文件中同一日期出现多条不同类型时，因无时长无法判定主运动；导入预览必须提示冲突，用户需选择保留项后才能完成导入。
- 每次成功导入创建一个独立导入批次。用户可以查看批次名称、导入时间、影响日期数，并删除整个批次；删除后所有受影响年份重新计算。
- 解析预览必须显示：总行数、成功行数、无效日期、未识别类型、重复日期、与 HealthKit 冲突的日期数。

## 5. 运动大类映射

HealthKit 或 CSV 中的细粒度运动类型先映射到大类，再参与日级与年级统计。映射字典须可维护，以便新增 Apple 健康类型时只补充配置。

| 运动大类 | 典型包含项 |
| --- | --- |
| 力量 | HIIT、传统力量训练、功能性力量、CrossFit、核心训练、混合力量训练。 |
| 跑步 | 户外跑、室内跑、跑步机、越野跑、田径跑。 |
| 骑行 | 户外骑行、室内骑行、动感单车。 |
| 游泳 | 泳池游泳、开放水域游泳。 |
| 球类 | 足球、篮球、羽毛球、网球、乒乓球、排球、棒球、橄榄球、壁球等。 |
| 板类 | 滑板、陆冲板、冲浪、单板滑雪等。 |
| 户外 | 徒步、步行、登山、攀岩、滑雪、越野、皮划艇等。 |
| 身心训练 | 瑜伽、普拉提、太极、拉伸训练。 |
| 舞蹈与操课 | 舞蹈、有氧操、健身操。 |
| 格斗 | 拳击、武术、跆拳道、搏击类。 |
| 划船 | 户外划船、室内划船机。 |
| 未映射类型 | 未配置或无法识别的 Workout 类型。 |

未映射类型在内部仍保留原始文本；在年度图片中，只要没有进入年度前四，都会和其余非前四类别一并归入“其他”。

## 6. 年历统计规则

### 6.1 日级归并

处理顺序如下：

1. 读取有效 HealthKit Workout 与已启用 CSV 批次。
2. 将每条记录映射到运动大类。
3. 按“日期 + 运动大类”累加 HealthKit 时长。
4. 选择当天累计时长最长的大类作为每日主运动。
5. 若时长相同，比较活动能量；仍相同则按固定大类优先级稳定决策。
6. CSV 覆盖模式启用时，该日以 CSV 类型映射后的大类为主运动；补全模式仅在该日不存在有效 HealthKit 主运动时生效。

示例：同一天有户外跑步 25 分钟、室内跑步 20 分钟、力量训练 35 分钟。映射并累计后，跑步共 45 分钟，因此该日显示“跑步”。

### 6.2 年级归并

1. 在每个自然年内，统计每种“每日主运动”出现的活动日数。
2. 按活动日数降序排序，取前四个运动大类。
3. 年度前四各自显示；其余所有类别合计显示为第五项“其他”。
4. 图例固定为五项：前四按数量降序，“其他”始终位于第五位。
5. 五项活动日数之和必须等于该年度总活动日数。

例如某年的排名为力量 128 天、跑步 67 天、游泳 41 天、球类 24 天，其余类别合计 31 天，则图例为：力量、跑步、游泳、球类、其他。

### 6.3 分类颜色

- 每个运动大类有稳定的品牌颜色；同一类别在不同年份进入前四时保持同色。
- “其他”始终使用统一辅助色。
- 一张年度卡片只出现五种类别色：年度前四和其他。
- 若某类在某年未进入前四，则该年归入其他；下一年重新进入前四时恢复其固定颜色。

## 7. 年度运动年历

### 7.1 网格布局

- 每列代表一周。
- 每列自上而下为周一至周日。
- 年初和年末不属于该自然年的日期显示为空白格。
- 一年根据日历自然生成 52–54 列。
- 月份标签位于该月第一天所在周的上方。
- 左侧显示简化星期标签，例如 Mon / Wed / Fri / Sun。
- 无活动日使用浅灰色；活动日使用每日主运动对应的年度颜色。
- 点击日期可查看当天采用的数据来源、主运动类型以及相关 Workout 列表；该详情不影响年历统计。

### 7.2 标题与统计

往年卡片标题格式：

```text
{用户名称} 的 2025 运动年历
251 / 365 活动日 · 68.8%
```

- 完整年份的分母为 365 或 366。
- 当前年份默认以“截至今天的自然日数”为分母，避免未来日期降低完成率。
- 标题、月份、星期和图例需要支持系统语言本地化。

## 8. 多年长图与导出

### 8.1 多年长图

- 用户可选择单年、多个指定年份或全部年份导出。
- 子图按年份倒序排列，最新年份位于最上方。
- 整张图顶部仅显示一次总标题，例如“Strivory · 我的运动年历”。
- 每张年度子图独立保留标题、统计、网格与五项图例。
- 最新年份为主视觉，完整显示；越早的年份透明度逐级增加。

推荐的不透明度阶梯：

| 相对最新年份 | 年度子图不透明度 |
| --- | ---: |
| 最新 | 100% |
| 前 1 年 | 82% |
| 前 2 年 | 67% |
| 前 3 年 | 55% |
| 前 4 年 | 45% |
| 更早 | 最低 38% |

透明度应用于子图的色块、文字与卡片层级；整张图片背景保持稳定，以保证导出可读性。

### 8.2 图片导出

- 输出格式：PNG。
- 模式：单年卡片、指定年份长图、全部年份长图。
- 输出应使用至少 3× 分辨率，适合相册、社交分享和打印。
- 支持系统分享面板：保存图片、存储到文件、AirDrop 与第三方分享目标。
- 导出前可预览；默认浅色主题，可选择深色主题。
- 第一版一次最多导出 10 年；超过时提示用户拆分导出。
- 可选显示低调的 `Made with Strivory` 标记，默认不影响主体阅读。

## 9. 页面与交互

### 9.1 首次使用与健康授权

- 解释仅读取 Workout，用于在本机生成运动年历。
- 请求 HealthKit 读取授权。
- 授权失败或未授权时可展示示例年历，并提供重新授权入口。

### 9.2 年度总览

- 年份切换。
- 年度活动日数与比例。
- 年度网格与五项图例。
- 点击日期查看详情。
- 刷新 HealthKit 数据。
- 导出当前年度或打开多年导出选择器。

### 9.3 多年回顾与导出预览

- 多选年份与“全部年份”快捷入口。
- 始终按最新到最早排序。
- 预览透明度层级、标题、统计和图例。
- 生成并调用系统分享。

### 9.4 CSV 导入

- 提供 CSV 模板下载。
- 使用系统文件选择器选取文件。
- 显示解析与冲突预览。
- 选择“补全缺失日”或“CSV 覆盖同日记录”。
- 确认导入后刷新年历。
- 管理、查看与删除历史导入批次。

### 9.5 设置

- 用户名称与导出标题。
- 时区与语言。
- HealthKit 重新授权。
- 导入批次管理。
- 导出主题与品牌标记开关。

## 10. 异常与空状态

| 场景 | 预期行为 |
| --- | --- |
| 未授权 HealthKit | 展示授权引导，不将其误报为“没有运动”。 |
| 某年无记录 | 展示空年历与“导入 CSV / 选择其他年份”入口。 |
| CSV 格式错误 | 不导入，指出错误行和错误原因。 |
| CSV 类型未知 | 在预览中显示原始文本；允许用户确认后归入其他。 |
| 当日 CSV 重复 | 要求用户选择保留类型后再导入。 |
| CSV 与 HealthKit 冲突 | 清晰展示日期、两种类型和所选导入策略。 |
| 导出失败 | 保留预览，允许重试，不丢失用户的年份选择。 |

## 11. 隐私与数据原则

- HealthKit 仅在用户授权后读取。
- 原始 Workout、CSV 文件内容、日级汇总和导出记录默认仅存储在设备本地。
- 不上传训练数据，不建立强制账号体系。
- 删除 CSV 导入批次后，相关本地记录与统计结果立即移除并重新计算。
- 分享操作仅导出用户确认生成的图片，不自动分享任何健康数据。

## 12. 第一版验收标准

1. 在 HealthKit 授权后，App 能读出有效 Workout，并忽略时长少于 10 分钟的记录。
2. 室内跑与户外跑在统计中统一计入“跑步”；泳池与开放水域统一计入“游泳”。
3. 同日多项运动按映射后的累计时长选择唯一主运动。
4. 每个年度卡片仅显示前四运动大类和“其他”，共五项。
5. “其他”包含当年全部未进入前四的已识别或未识别运动类型。
6. 周列从周一到周日纵向排列，年外日期为空，网格随年份正确生成 52–54 列。
7. 最新年份位于多年长图顶部，越早年份的视觉透明度越高。
8. 用户可导入符合格式的 CSV，查看预览，并按补全或覆盖规则影响年历。
9. 用户可删除一个 CSV 导入批次，受影响年份随即恢复为删除后的正确状态。
10. 用户可导出单年或最多 10 年的高清 PNG，并通过 iOS 系统分享面板保存或分享。
